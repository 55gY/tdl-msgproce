# Copilot 编码指南

本项目使用以下编码规范：

## 日志输出规范

- **可视化日志**: 使用 `fmt.Printf()` 输出用户可见的日志信息
- **调试日志**: 使用 `// fmt.Printf()` 注释掉的调试日志
- **不使用**: ~~zap日志库~~（已移除）

## 日志示例

```go
// ✅ 正确 - 用户可见的重要信息
fmt.Printf("✅ 服务启动成功\n")
fmt.Printf("📨 收到新消息: ID=%d\n", msgID)

// ✅ 正确 - 调试信息（已注释）
// fmt.Printf("[DEBUG] 处理消息: %+v\n", msg)
// fmt.Printf("[DEBUG] 配置加载: %s\n", configPath)

// ❌ 错误 - 不要使用 zap
// log.Info("message")
// p.ext.Log().Info("message")
```

## 代码风格

1. 所有面向用户的输出使用 `fmt.Printf`
2. 调试和详细日志应注释掉，保留 `// fmt.Printf` 格式
3. 使用 emoji 增强日志可读性：✅❌⚠️📨🔗💡等
4. 保持日志简洁，关键信息突出

## 变更管理规范

**每次代码调整后，必须更新对应的说明文件**：

1. **文档更新要求**：
   - 在完成代码调整后，立即更新相关的说明文档
   - 记录本次调整的目的、影响范围和关键变更
   - 确保文档与代码保持同步

2. **说明文件示例**：
   - 功能更新 → 更新 `README.md`
   - 日志系统变更 → 更新 `LOGGING_UPDATE_STATUS.md`
   - 文件上传功能 → 更新 `FILE_UPLOAD_GUIDE.md`
   - 构建流程变更 → 更新 `BUILD.md`

3. **记录格式建议**：
   ```markdown
   ## [日期] 变更说明
   - **调整内容**: 简述本次调整
   - **影响范围**: 说明影响的模块/功能
   - **注意事项**: 使用时需要注意的要点
   ```
